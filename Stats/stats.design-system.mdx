
# ðŸ“’ Developer style guide for Stats
Stats is something is like all the numeric data. So, In web-pages the stats is GUI control component that help users to easily visiualise data related to it apart from their background.

We represent stats with the help of store button that includes the data and related icon. It is versetile in nature means this component can easily used anywhere in webpages. Some of the common example as what is monthly download, engagement of users, numbers of subscribers etc. Even in the codepeak program there is use case of stats as number of contributor, number of pull request.

![image](https://i.ibb.co/wYm5kzM/Screenshot-2311.png) ![image](https://i.ibb.co/tBBpxRF/Screenshot-2312.png)

## Types of stats ðŸ”½

1. Icon stats
2. shared bodered stats
3. Graphical stats

---

## General must known design system guideline :

### Color ðŸŽ¨

| Background | Yellow | White | Dark-Gray | Green | Blue | Red|
| --- | --- | --- | --- | --- | --- | --- |
| Code in Tailwind | bg-yellow-500 | bg-white | bg-gray-600 | bg-Green-500| bg-Blue-500| bg-Red-500|

Color of `bg` is generally used as shade of 100 / 700 

Color of `text` will depend on the `bg` color
| Background | Dark | Light |
| --- | --- | --- |
| Text |  Dark | Light  |
| Code in Tailwind | text-white,text-gray-600 | text-green-600,text-grey-600,text-blue-500 |

`Note` : Color of the text `[color]` will depend on the color of the background

### Font

For all stats font is : **`font-medium font-bold`**

### SVG And Path
SVG stands for Scalable Vector Graphic. It can be used to make graphics and animations like in HTML canvas.  Path is always used within the svg tag to set the actual coordinate.
Wihin the tag we give the some propety as what with be the co-ordinate, fill color, outer stroke, and some styling within class attribute.

### Border Radius ðŸ”²

The normal border-radius for the components are `rounded-md rounded-lg`. However this does not apply in many cases like buttons, input boxes and all.

---

## Icon guidelines:

Mostly all the icons are taken from â†’ [Hero Icons](https://heroicons.com/)

`Icon-text-gap` :  We follow a convention of equal margin between the text and the icon as padding in y  axis `py`. and always the margin is given from the side of icon.

|  | Width | Type |
| --- | --- | --- |
| Small | w-4 & h-4 | Hero Icons mini |
| Medium | w-5 & h-5 | Hero Icons mini |
| Large | w-6 & h-6 | Hero Icons filled |
